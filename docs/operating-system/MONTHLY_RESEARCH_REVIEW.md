# Monthly Research Review

Run this review every 4 weeks.

## Inputs

- paper notes completed this month;
- bilingual glossary updates;
- TRI experiment or critique notes;
- technical-report draft sections;
- unresolved questions.

## Review Areas

### Reading

- How many papers were completed?
- Which paper was hardest?
- Which paper was most reusable?
- Which English terms are still blocking comprehension?

### Research Judgement

- Can the researcher distinguish method, experiment, and claim?
- Can the researcher explain what a paper does not prove?
- Can the researcher identify weak baselines or weak metrics?

### Engineering

- Did reading lead to any TRI experiment, critique, or implementation idea?
- Were any results reproducible?
- Were notes connected to actual artifacts?

### Writing

- Which technical-report section improved?
- Which paragraph still overclaims?
- Are limitations written clearly?

## Monthly Decision

End with one of:

- continue current track;
- narrow the topic;
- add a stronger evaluation focus;
- add more English-reading training;
- start Agent Runtime as a secondary track;
- pause new papers and consolidate notes.
