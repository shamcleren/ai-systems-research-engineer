# Reading Tracks

This file defines reading tracks, not a fixed paper list. Specific papers should be selected during weekly planning so the list can stay current and match the researcher's level.

## Track A: Paper Reading On-Ramp

Purpose: learn how AI systems papers are structured.

Paper types:

- survey or tutorial-style AI systems papers;
- technical reports with clear structure;
- system papers with readable architecture diagrams.

Output:

- structured Chinese note;
- bilingual keyword table;
- one system diagram redrawn in text or Mermaid;
- one paragraph on what the paper does not prove.

## Track B: Context Engineering

Purpose: understand how systems construct useful context for models, tools, and humans.

Paper themes:

- retrieval-augmented generation;
- context compression;
- long-context management;
- memory systems;
- provenance-preserving context;
- structured context representations.

Output:

- context construction pattern summary;
- comparison with TRI Context Package;
- one reusable design idea.

## Track C: AI Evaluation

Purpose: learn how evidence is built and bounded.

Paper themes:

- RAG evaluation;
- agent evaluation;
- benchmark design;
- human and model judging;
- synthetic and replay-style evaluation;
- ablation and stress testing.

Output:

- metric table;
- baseline critique;
- threats-to-validity notes;
- claim boundary audit.

## Track D: Agent Runtime, Later

Purpose: delayed track after Context Engineering and Evaluation foundations are established.

Paper themes:

- tool-use loops;
- planning and execution;
- state management;
- multi-agent coordination;
- runtime observability.

Entry condition:

- at least 8 completed structured paper notes;
- at least 2 evaluation critiques;
- one technical-report draft section.
