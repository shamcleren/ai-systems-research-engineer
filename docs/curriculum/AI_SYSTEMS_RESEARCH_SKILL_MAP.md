# AI Systems Research Skill Map

## Track 1: Paper Reading Foundation

Goal: build the ability to read, summarize, and critique AI systems papers.

Core skills:

- identify problem statement;
- distinguish motivation, method, experiment, and claim;
- extract system architecture;
- identify assumptions and non-claims;
- summarize contribution in one sentence;
- build an English-Chinese keyword glossary.

Exit criteria:

- read one AI systems paper per week with structured notes;
- explain the paper's contribution and limitation in Chinese;
- extract at least 15 useful bilingual terms per paper.

## Track 2: Context Engineering

Goal: understand how AI systems select, package, compress, preserve, and evaluate context.

Core topics:

- retrieval and context selection;
- context compression;
- memory and state;
- provenance and citation;
- prompt/context construction;
- structured context packages;
- context boundaries for downstream models, tools, or humans.

Exit criteria:

- define context engineering in the researcher's own words;
- compare at least three context construction patterns;
- connect each pattern to TRI's Context Package design.

## Track 3: AI Evaluation

Goal: learn how to design and critique evaluation for AI systems.

Core topics:

- benchmark design;
- oracle construction;
- synthetic vs replay-style evaluation;
- coverage, precision, recall, noise, compactness;
- ablation studies;
- stress cases;
- validity threats;
- claim boundaries.

Exit criteria:

- critique an evaluation table without only reading the numbers;
- identify what an experiment supports and what it does not support;
- write a threats-to-validity section for a technical report.

## Track 4: Research Engineering

Goal: turn research ideas into reproducible engineering artifacts.

Core topics:

- minimal prototype design;
- schema and data model boundaries;
- experiment scripts;
- stable result artifacts;
- deterministic tests;
- documentation tied to code and experiments.

Exit criteria:

- design a small experiment from a hypothesis;
- produce reproducible commands and outputs;
- keep code, experiment notes, and report text consistent.

## Track 5: Academic Writing

Goal: write paper-style technical reports with clear contributions and conservative claims.

Core topics:

- abstract;
- introduction;
- problem definition;
- method;
- experimental setup;
- results;
- limitations;
- related work;
- artifact appendix.

Exit criteria:

- write a complete technical-report section each week after week 5;
- distinguish claim, observation, hypothesis, and future work;
- revise text to avoid overclaiming.
