# Roadmap Image Prompt / 路线图图片 Prompt

## Purpose / 目的

中文：这张图用于帮助快速理解整个 12 周训练路线：从论文阅读入门，到 Context Engineering，再到 AI Evaluation，最后形成 technical report artifact。

English: This image helps communicate the 12-week training path: from paper-reading foundation, to Context Engineering, to AI Evaluation, and finally to a technical-report artifact.

## Prompt / 生成 Prompt

Create a clean bilingual roadmap infographic for a personal AI Systems Research Engineer growth plan.

Audience: a senior infrastructure engineer who is a beginner in academic AI systems research.

Format: horizontal roadmap, 16:9, high readability, suitable for a README or learning plan.

Visual structure:
- Four large phases from left to right.
- Phase 1: "Weeks 1-3 / 第 1-3 周: Paper Reading Foundation / 论文阅读基础"
- Phase 2: "Weeks 4-6 / 第 4-6 周: Context Engineering / 上下文工程"
- Phase 3: "Weeks 7-9 / 第 7-9 周: AI Evaluation / AI 评测方法"
- Phase 4: "Weeks 10-12 / 第 10-12 周: Technical Report / 技术报告产出"

Under each phase, show 3 compact bullets:
Phase 1:
- "Read structure / 拆论文结构"
- "Build glossary / 建术语表"
- "Claim vs evidence / 区分论断与证据"
Phase 2:
- "Context package / 上下文包"
- "Retrieval and memory / 检索与记忆"
- "TRI connection / 连接 TRI"
Phase 3:
- "Metrics / 指标"
- "Baselines / 基线"
- "Validity threats / 有效性威胁"
Phase 4:
- "Problem definition / 问题定义"
- "Method and results / 方法与结果"
- "Limitations / 局限性"

Add a bottom loop labeled:
"Weekly loop / 每周闭环: Read -> Translate -> Extract -> Evaluate -> Connect to TRI -> Write"

Style:
- professional research notebook aesthetic;
- white or very light background;
- restrained colors: deep blue, teal, charcoal, and warm amber accents;
- no decorative blobs or gradients;
- use simple line icons: book, package, chart, document;
- crisp typography;
- all text must be legible and not overlap;
- avoid tiny text;
- no logos, no watermark.
